Project Goals

Initially, I didn't set out to make a full command and control framework, capable of managing multiple implants at once and modularly loading new payloads. At first, I just wanted to make a rust based RAT, or backdoor, since I couldn't find any quality open source options. However, what started as a small backdoor spiraled into a full C2 apparatus. My only true goal with this project is to make a unique and creative C2 framework. I chose WASM due to its portability and isolated runtime environment, making it perfect for an implant system.

Bird's Eye View

This entire page goes into depth about most design decisions, but the TL;DR would be:

  1. The initial implant is loaded onto target system
  2. Rust binary is executed, WASM runtime environment is initialized
  3. WASM runtime listens for server over encrypted UDP channel, server sends sharded WASM binaries that are in accordance with the ABI
  4. Payloads are run using minimal system resources and no footprint. Everything is fully in memory, avoiding disk writes and persistent artifacts
  5. Sends sharded data back over UDP channel using jitters and randomized timing
