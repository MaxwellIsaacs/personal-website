Project Overview

The MOS 6502 powered some of the most iconic computers of all time: the Apple II, Commodore 64, Nintendo Entertainment System, and BBC Micro. This emulator recreates every aspect of the original hardware, from instruction timing to memory mapping, providing an authentic platform for running vintage software and understanding computer architecture fundamentals.

Technical Implementation

Cycle-Accurate Timing

Every instruction executes in the exact number of clock cycles as the original hardware, including page boundary crossing penalties and branch timing variations.

Complete Instruction Set

All 151 documented and undocumented opcodes implemented, including the infamous "illegal" instructions used by some games and demos.

Memory Mapping

Accurate memory layout with zero page, stack, and memory-mapped I/O regions. Supports bank switching and custom memory configurations.

Integrated Assembler

Built-in assembler with label resolution, macro support, and comprehensive error reporting. No external tools required.

Interactive Debugger

Step-by-step execution, breakpoints, register inspection, and memory dumping. Perfect for understanding program flow and debugging.

Zero Dependencies

Pure Rust implementation with no external crates. Self-contained and portable across all platforms that support Rust.

Architecture & Design

The emulator is structured around the core components of the 6502 architecture, each implemented as separate modules that interact through well-defined interfaces.

CPU Core

The central processing unit handles instruction decoding, execution, and state management. Each addressing mode is implemented separately for clarity and maintainability.

  • 8-bit accumulator and index registers
  • 16-bit program counter and stack pointer
  • Status register with all flags (N, V, B, D, I, Z, C)
  • 13 addressing modes including indexed indirect

Memory System

64KB linear address space with special regions for system functionality and peripheral communication.

  • Zero page ($0000-$00FF) for fast access
  • Stack at $0100-$01FF (256 bytes)
  • Memory-mapped I/O regions
  • Configurable ROM/RAM layouts

Applications & Learning

This emulator serves multiple purposes beyond just running vintage software. It's an educational tool for understanding computer architecture and a development platform for 6502 programming.

Development

Run historical software and understand how classic computers operated. Preserve computing history for future generations.
